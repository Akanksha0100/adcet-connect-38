# Contributing

Thank you for your interest in the **ADCET Alumni Portal**. We genuinely
appreciate the time and curiosity it takes to look through the codebase.

## A note on external contributions

This project is an institutional product maintained internally for ADCET.
At this time we **do not accept code contributions, pull requests, or
patches from individuals or organisations outside the maintaining team**.
Any PRs opened against this repository will be respectfully closed without
a code review.

This isn't about the quality of your work — it's purely about the
governance, security review, and academic-policy constraints that come
with running a portal tied to real student and alumni records.

## Found a bug or have a feature idea? — please open an issue

We *do* warmly welcome:

- 🐞 **Bug reports** — something broken, confusing, or visually off
- 💡 **Feature requests** — a workflow that would help alumni / students / admins
- 🔐 **Security disclosures** — please mark these clearly and avoid public PoCs
- 📚 **Documentation gaps** — README sections that misled you

When you open an issue, the more context you can give us, the faster we
can act on it:

1. **What you were trying to do** — the goal, not just the click path
2. **What happened** vs. **what you expected**
3. **Steps to reproduce** (smallest possible recipe)
4. **Environment** — browser / OS / role you were logged in as
5. **Screenshots, logs, or HAR files** when relevant (please redact PII)

## Code of conduct

Be kind, be specific, assume good intent. Issues that are abusive, spammy,
or off-topic will be closed without further comment.
